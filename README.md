# Failed Order Analysis

## Project Overview

This project analyzes failed ride orders to identify patterns behind order failures and uncover actionable business insights.

Using exploratory data analysis (EDA), data visualization, and statistical analysis, the project investigates:

- Peak hours of order failures
- Driver cancellation trends
- No-driver-assigned incidents
- Failure distribution across time
- Operational bottlenecks affecting order completion

---

## Business Problem

Ride-hailing platforms often experience failed orders due to:

1. No driver being assigned
2. Driver cancellations after assignment

Understanding when and why these failures occur can help improve customer experience and operational efficiency.

## Key Insights

### Hourly Failure Trends
- Certain hours experience significantly higher order failures.
- Peak demand periods show increased operational stress.

### Driver Availability Issues
- A large proportion of failures occur due to no driver assignment.

### Cancellation Patterns
- Driver cancellations tend to cluster during specific time windows.
